Phase 3 — Persist (from Approach B)

Approved map saved as story-map.md with checkboxes

Phase 4 — Implement, sticky by sticky (from Approaches A + C)

Each session: skill reads map → user picks next sticky → implementation agent fires

Auto-invokes /hexagonal · TDD · /refactor · commits · checks off sticky · repeat

What each approach contributes

B — Persistent Map Story map as living .md file — resumable, version-controlled, always the source of truth
C — Three Agents Drafter builds the map, Provocateur challenges it, focused agent implements each sticky
