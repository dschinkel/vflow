Living Story Map UI — Options

From simplest to most powerful. All are buildable now.

A

Extend the Brainstorm Server (Zero Setup)

We already have a local server running. The skill generates a story-map.html that renders like a board. Agent updates the file as stickies complete — browser auto-refreshes. No React, no build step, works today.

B

Tiny React/Vite App (Scaffolded by the Skill)

The /feature skill scaffolds a small story-map-app/ alongside your project. Skill writes to story-map.json; the React app reads it and renders the board live (hot-reload). Can add drag-to-reprioritize, filters, persona toggle. Shareable on a local URL.

story-map-app/ (Vite dev server)
story-map-app/
  src/
    StoryMap.tsx    ← board component
    Sticky.tsx     ← card component
    main.tsx
  story-map.json  ← agent writes here
  package.json

Pros

  • Real interactive UI
  • Drag/drop possible
  • Extendable over time
  • Shareable locally

Cons

  • Needs npm install
  • Extra app to maintain
  • Build step to deploy
C

Miro via REST API

The skill pushes stickies directly to a Miro board via their REST API. As the agent works, stickies move from "To Do" to "Done" lanes in real Miro. Shareable with teammates, looks exactly like a real story map board.

Miro Board — live, shareable URL
Skill calls → POST /v2/boards/:id/sticky_notes
Agent completes sticky → PATCH to move it to Done lane
Team sees live board at miro.com/…

Pros

  • Real Miro board
  • Team-shareable
  • Full Miro features

Cons

  • Requires Miro account + API key
  • External dependency
  • Rate limits

My take

Start with A to validate the flow fast — zero friction, works today. Build B (React app) as the next iteration once the skill flow is solid — it becomes the real living UI. C (Miro) is the power move for team use, but save it for when you want to share with others.
