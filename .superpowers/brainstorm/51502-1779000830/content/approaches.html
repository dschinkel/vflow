Feature Workflow Skill — Three Approaches

These are the three ideas from our earlier discussion. Select the one(s) that resonate — or mix them.

A

Conductor Skill /feature

One skill runs the full XP flow in a single session: Value Story → Story Map → approve → implement stickies one at a time. Each sticky auto-invokes the right sub-skills (hexagonal, TDD, refactor). Commits after every sticky. The story map file IS the backlog.

Pros

  • Single entry point
  • Fully guided end-to-end
  • Sub-skills fire automatically

Cons

  • Long sessions
  • Harder to resume mid-flow
B

Persistent Story Map

Run /feature once to create and approve the story map as a markdown file with checkboxes. Every future session the skill reads the map, finds the next unchecked sticky, and implements it. Progress survives across sessions. Story Map = review. Feature Archive = memory.

Pros

  • Resumable across sessions
  • Map is always the source of truth
  • Fits long-running features

Cons

  • Needs map file management
  • Two invocations to start
C

Three-Agent Structure

Separate agents for each role: Drafter produces the story map, Provocateur challenges assumptions before you approve, then one focused Implementation agent per sticky using all the sub-skills. Adversarial check built in before any code is written.

Pros

  • Quality gate before coding
  • Each agent is focused
  • Disagreement surfaces blind spots

Cons

  • More orchestration overhead
  • Heavier to start

My recommendation

Combine A + B: the conductor flow for creation, the persistent map for resumability. Add a lightweight Provocateur step (not a full agent) as a challenge gate before the map is approved. C's three-agent structure is better saved for a later iteration once the core flow is solid.
